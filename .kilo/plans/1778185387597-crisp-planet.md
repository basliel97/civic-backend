# Plan: Add Notifications for Report Resolutions

## Goal
Implement notification triggers in the report service to inform reporters when their content reports are resolved or rejected by moderators.

## Analysis
- The report service (`src/services/report.ts`) has `resolveReport` and `rejectReport` functions that update report status.
- Currently, no notifications are sent to the reporter about the outcome.
- Adding notifications will improve user experience by keeping reporters informed of moderation actions.

## Implementation Steps

1. **Import Notification Function**
   - Add import for `notifyUser` from `./agency.js` to `src/services/report.ts`

2. **Modify resolveReport Function**
   - After successful database update, fetch the reporter's user_id from the returned report data
   - Send notification with title "Report Resolved", message including the resolution text, type "success", screen "report_detail", targetId report.id
   - Wrap notification call in try/catch to prevent failures from crashing the function

3. **Modify rejectReport Function**
   - After successful database update, fetch the reporter's user_id from the returned report data
   - Send notification with title "Report Rejected", message "Your report has been reviewed and rejected", type "warning", screen "report_detail", targetId report.id
   - Wrap notification call in try/catch to prevent failures from crashing the function

4. **Testing**
   - Verify notifications are sent correctly without affecting main functionality
   - Test both resolve and reject scenarios
   - Ensure try/catch blocks log errors appropriately

## Safety Considerations
- All notification calls wrapped in try/catch blocks
- Main database operations are not affected by notification failures
- No changes to existing function signatures or return values

## Files to Modify
- `src/services/report.ts`: Add import and notification logic to resolveReport and rejectReport functions
